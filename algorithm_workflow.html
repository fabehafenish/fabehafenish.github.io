Phase 1 — Quick Lesson Planning

Steps 1–3 · 15–20 minutes
MagicSchool AI
1

Generate Core Lesson Plan

MagicSchool AI
Prompt Used Create a Grade 3 Primary ICT lesson on "What is an Algorithm?" for a Cambridge IGCSE Primary curriculum class. Include: 3 clear learning objectives, a 5-minute starter activity, a 15-minute main activity, a formative assessment check, and a 5-minute plenary/reflection. Align with a mixed-ability classroom.
✦ MagicSchool AI Output — Core Lesson Plan

📚 Learning Objectives

  • Define the term algorithm and identify it as a step-by-step set of instructions used to solve a problem.
  • Create a simple algorithm for an everyday activity (e.g. making a sandwich, brushing teeth) using sequential steps.
  • Identify errors in a given algorithm and suggest corrections, developing basic debugging thinking.

🌅 Starter Activity (5 mins) — "Robot Instructions"

Teacher gives a sequence of actions and students decide if they are in the correct order. Example: "Put on shoes → Tie laces → Put on socks" — class identifies the error and corrects the sequence. Whole-class verbal activity, no materials needed.

⚙️ Main Activity (15 mins) — "My Algorithm Recipe"

Students write a step-by-step algorithm to make a jam sandwich. They number each step and use sequence (first, next, then, finally) language. In pairs, they swap and test each other's algorithm by role-playing as a literal robot — following instructions exactly as written to discover missing or incorrect steps.

📋 Formative Assessment Check

Exit ticket: Students draw a 4-step flowchart for "Getting ready for school in the morning" using arrows to show sequence. Teacher reviews while students complete a plenary activity.

💬 Plenary / Reflection (5 mins)

Class discusses: "Can a computer follow an algorithm? Why does every step need to be clear?" Teacher introduces the connection to programming — computers follow algorithms written as code, and a missing step causes an error (bug).

3

Differentiation for Mixed Abilities

MagicSchool AI
Prompt Used Modify this algorithm lesson for: (1) low ability learners who need simplified tasks, (2) ESL learners who need basic English vocabulary, and (3) advanced learners who need extension/challenge tasks.
✦ Differentiated Plans

🔵 Low Ability Learners

  • Pre-cut sentence strips — students sequence rather than write steps
  • Algorithm limited to 3 steps maximum
  • Worked example provided with first step completed
  • Peer support encouraged; word bank provided
  • Exit ticket: match step to picture instead of drawing flowchart

🟢 ESL Learners

  • Visual flowchart template with picture cues (icons for each step)
  • Sequence word cards: First / Next / Then / Finally in English + home language reference
  • Algorithm task uses familiar physical actions (no cultural assumptions)
  • Partner talk allowed in home language before recording in English

🟣 Advanced Learners

  • Write an algorithm for a more complex task (e.g. "Log in to the computer and open a file")
  • Introduce the concept of conditional steps: "IF the sandwich has no jam, THEN add jam"
  • Attempt to identify a bug in a given incorrect 6-step algorithm
  • Extension: Explain in writing why the order of steps matters in an algorithm

4

Worksheet — 5 MCQs + 3 Short Answer + 1 HOT Question

Brisk Teaching
Prompt Used Create a worksheet based on the "What is an Algorithm?" Grade 3 ICT lesson. Include: 5 multiple choice questions, 3 short answer questions, and 1 higher-order thinking question. Align with Cambridge IGCSE Primary ICT learning outcomes.
✦ Brisk Teaching Output — Student Worksheet

Part A — Multiple Choice Questions (Circle the correct answer)

1. An algorithm is best described as:

A) A type of computer B) A step-by-step set of instructions ✓ C) A drawing of a robot D) A computer game

2. Which of the following is an example of an everyday algorithm?

A) Playing outside B) Steps to brush your teeth ✓ C) Choosing a colour D) Watching a film

3. Why must the steps in an algorithm be in the correct order?

A) To make it look pretty B) So a computer can print it C) So the task is completed correctly ✓ D) To save electricity

4. A mistake in an algorithm is called a:

A) Step B) Code C) Bug ✓ D) Loop

5. Algorithms are used in computers because:

A) Computers can talk B) Computers like lists C) Computers follow exact instructions to complete tasks ✓ D) Computers are very fast readers

Part B — Short Answer Questions

1. Write the steps to make a cup of tea in the correct order. Use the words: First, Next, Then, Finally.

Expected answer: First, boil the water. Next, place a teabag in the cup. Then, pour in the hot water. Finally, add milk and stir.

2. Look at this algorithm: (1) Put on shoes → (2) Put on socks → (3) Walk to school. What is wrong? How would you fix it?

Expected answer: The order is wrong — socks should come before shoes. Fix: (1) Put on socks → (2) Put on shoes → (3) Walk to school.

3. Give one reason why algorithms are important for computers.

Expected answer: Computers cannot think for themselves. They need clear, ordered instructions (algorithms) to know what to do and how to complete a task correctly.

Part C — Higher-Order Thinking Question

HOT — Evaluate & Create

Your friend says: "Algorithms are only for computers — humans don't use them." Do you agree or disagree? Give TWO examples from real life to support your answer, then write your own 5-step algorithm for an activity of your choice.

Expected response indicators: Disagreement; examples from daily life (cooking recipe, school morning routine, sport instructions); original algorithm with 5 correctly sequenced steps, using order words; recognition that humans follow algorithms all the time without realising it.

6

Engagement Activity — Think-Pair-Share

Brisk Teaching
Prompt Used Convert this lesson into a Think-Pair-Share activity with 2 discussion questions and 1 real-life application example.
✦ Think-Pair-Share Activity

Discussion Questions

  • Think: Can you think of something you do every day that has steps that always happen in the same order? Write down 3 steps. (2 mins silent thinking)
  • Pair: Share your sequence with your partner. Does your partner agree the order is correct? Could you swap any steps without breaking the process? (2 mins)

Real-Life Application Example

A map app on a phone (like Google Maps) uses an algorithm to find the fastest route to school. It checks every possible road, calculates the time for each, and picks the shortest path — all in less than one second. Ask students: "What would happen if the app got the steps in the wrong order?"

Automated Feedback — Sample Student Response

Microsoft Copilot
Prompt Used (anonymous response) Provide constructive feedback for this student answer: "An algorithm is list of things to do. My sandwich: 1. Put bread on plate 2. Add jam 3. Put second bread on top." Identify: Strength, Area to improve, Suggested correction.
✦ Copilot Output — Student Feedback
💪 Strength

You understand the key idea — an algorithm is a set of instructions, and you've shown the steps in a logical order. Your sandwich algorithm works and has a clear beginning and end. Well done for understanding that an algorithm needs multiple steps!

📈 Area to Improve

Your algorithm is missing the sequence words (First, Next, Then, Finally) that help make the order very clear. The definition could be extended — an algorithm isn't just a list, it must be in a specific order to solve a problem correctly.

✏️ Suggested Improvement

Try rewriting it like this: "First, place one slice of bread on a plate. Next, spread jam on the bread. Then, place the second slice of bread on top. Finally, cut the sandwich in half." — Notice how adding the order words makes each step much clearer!

10

End-of-Day Reflection

Microsoft Copilot
Prompt Used Summarize: what worked well in this algorithm lesson, what needs improvement, and suggest 1 change for the next class.
✦ Copilot Output — Teacher Reflection

✅ What Worked Well

The "Robot Instructions" starter was highly engaging — students immediately connected the concept of wrong order to a concrete, funny scenario. The Think-Pair-Share activity generated strong verbal discussion and the sandwich analogy made the abstract concept accessible to all learners including EAL students.

⚠ What to Improve

The exit ticket flowchart took longer than expected — some students needed more support with the arrow/flowchart format. Low ability learners needed the pre-cut strips earlier in the lesson to stay engaged. Time on the main activity slightly overran due to the pair-swap testing element.

🔁 1 Change for Next Class

Introduce a simple digital algorithm activity using Code.org's "Hour of Code" to show students how their paper algorithm connects directly to block-based programming. This bridges the lesson into the next unit on programming and maintains engagement through hands-on computer interaction.
